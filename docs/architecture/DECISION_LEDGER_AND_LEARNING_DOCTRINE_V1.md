# TOKENOSKOBI DECISION LEDGER AND LEARNING DOCTRINE V1

## Amaç

Sistem sadece işlem yapan bir sistem olmayacaktır.

Sistem:

- Karar veren
- Kararları arşivleyen
- Sonuçları muhasebeleştiren
- Hatalardan öğrenen
- Kaçırılan fırsatları analiz eden
- Sürekli gelişen

bir harekât sistemi olacaktır.

---

# SENARYO 1 — GERÇEK İŞLEM

Token bulundu
→ Skorlandı
→ Giriş seviyesi üretildi
→ Stop Loss üretildi
→ Take Profit üretildi
→ Kullanıcı işleme girdi
→ İşlem sonuçlandı

Arşivlenecek:

- Token bilgileri
- Skorlar
- Giriş nedeni
- Risk nedeni
- Balina durumu
- Likidite durumu
- Haber durumu
- Teknik veriler
- Çıkış nedeni
- Kar/Zarar sonucu

---

# SENARYO 2 — PAPER ASSAULT

Token bulundu
→ Kullanıcı çevrimdışı
→ Paper Trade açıldı
→ Simüle giriş yapıldı
→ Simüle çıkış yapıldı
→ Sonuç hesaplandı

Amaç:

Kaçırılan fırsatları öğrenme verisine dönüştürmek.

---

# SENARYO 3 — REDDEDİLEN FIRSAT

Token bulundu
→ Skor düşük bulundu
→ İşleme girilmedi
→ Token izlenmeye devam edildi

Amaç:

İşleme girilmeyen fırsatlardan da öğrenmek.

---
