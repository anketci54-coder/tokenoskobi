## SECTION 2: PURPOSE

CASE_LIBRARY.md defines the constitutional governance framework for institutional case memory in Tokenoskobi. It governs how cases are formed, classified, linked, validated, retained, replayed, learned from, and constitutionally protected.

This document is enterprise constitutional documentation only. It includes no code, no APIs, no runtime logic, and no implementation detail.

The Case Library is a constitutional memory surface that supports:
- accountability permanence,
- specialist traceability,
- operator continuity,
- guardian compliance,
- controlled learning evolution.

## SECTION 3: CASE LIFECYCLE

### 3.1 Lifecycle States

Constitutional lifecycle states are:
1. **INITIATED**
2. **EVIDENCE_INTAKE**
3. **SPECIALIST_REVIEW**
4. **CONSENSUS_SYNTHESIS**
5. **DECISION_GATE**
6. **OUTCOME_TRACKING**
7. **LESSON_EXTRACTION**
8. **SEALED_ARCHIVE**

### 3.2 Lifecycle Governance

- Case lifecycle progression must be auditable.
- A case cannot skip constitutional states.
- Sealed cases are immutable except through constitutional amendment or explicit constitutional correction protocol.
- Lifecycle ownership remains explicit at every state transition.

### 3.3 Lifecycle Authority

- Guardian may hard-block non-compliant progression.
- Risk authority governs risk verdict constraints.
- Harekât Subayı governs synthesis orchestration.
- Operator governs human decision gate action within constitutional boundaries.

## SECTION 4: CASE CLASSIFICATION

### 4.1 Constitutional Classification Axes

Each case is classified across immutable governance axes:
- **Case Class:** Normal / Elevated / Critical / Emergency
- **Evidence Integrity Class:** Verified / Partial / Contested
- **Risk Class:** Low / Medium / High / Severe
- **Confidence Class:** Low / Guarded / Strong / High
- **Governance Class:** Compliant / Escalated / Guardian-Blocked
- **Outcome Class:** Approved / Rejected / Deferred / Escalated

### 4.2 Classification Discipline

- Classification must be explicit and explainable.
- Conflicting classifications must be recorded, not hidden.
- Classification changes must include reason and accountable owner.
